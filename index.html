6. Conclusión

¿Por qué?

  • Usar GitHub → Control de versiones, colaboración y backup del código
  • Usar patrones software → Código reutilizable, escalable y más fácil de mantener
  • Hacer un README → Facilita entender y usar el proyecto rápidamente
  • Hacer diagramas UML → Claridad en la estructura y flujo del sistema
  • Integración continua → Automatización de testing y despliegue
